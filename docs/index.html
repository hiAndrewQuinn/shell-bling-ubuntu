shell-bling-ubuntu

23 command-line tools, pinned. One wget. Tested on 30 distros every commit. fzf, fish, nvim+LazyVim, the whole modern-CLI starter pack — installed silently while you go get a coffee.

install 
curl -fsSL https://raw.githubusercontent.com/hiAndrewQuinn/shell-bling-ubuntu/main/install.sh | sh

No curl yet? (Fresh Debian 13 doesn't ship it.) wget -qO- https://raw.githubusercontent.com/hiAndrewQuinn/shell-bling-ubuntu/main/install.sh | sh works too — the script installs curl before it needs to fetch anything else.

For legacy-glibc distros, five Rust binaries (bat, fd, eza, lsd, starship) auto-route to their musl variants. You get the same version as on a modern distro, just statically linked.

Out of scope deliberately, not laziness — architectural mismatches with the "drop pinned binaries into /usr/local/bin" model
NixOS, GuixSDfunctional pkg mgmt — needs proper Nix expressions
CoreOS, Flatcar, Bottlerocket, ChromeOSimmutable; /usr is read-only
FreeBSD, OpenBSD, NetBSDdifferent OS family; most upstream binaries don't ship BSD variants
SLES 15official Docker requires SUSE subscription; use openSUSE Leap 15.6 instead

what the script does

1.
Detect platform. /etc/os-release, glibc version, architecture, virt status, init system. Probes sudo / doas / root with a usability test, not a "is it on PATH" test — so non-sudoers get a real error message instead of a wasted password prompt.
2.
Sudo password once. A background keepalive refreshes the timestamp every 50s so you're never prompted again mid-install.
3.
Install everything silently. Native packages where they exist; upstream-pinned tarballs (verified by sha256) for the rest. Parallel fetch with a progress ticker. Per-platform softening for ancient glibcs.
4.
Two fzf pickers. One for your default editor (nvim / vim / hx / micro). One for whether to chsh to fish. Skip both with SHELL_BLING_NONINTERACTIVE=1.
5.
Friendly recap, then exit. Log out of your desktop session and back in (or reboot) for $SHELL + the new x-terminal-emulator default to take effect. Just reopening a terminal isn't enough on most desktops.

try before you install

Drop into a fresh container with the installer already done — kick the tires without touching your real system.

git clone https://github.com/hiAndrewQuinn/shell-bling-ubuntu
cd shell-bling-ubuntu
make dev DISTRO=ubuntu-24.04   # or debian-13, ubuntu-22.04, etc.

honest disclaimers

Pre-1.0. Backwards-compatibility breaks are expected. Pin a commit if you care.

It's a productive-shell installer, not a language-toolchain manager. No Rust, no Go, no Python interpreter setup. Install those yourself from upstream after shell-bling runs.

The matrix is real. 30 distros in Docker on every commit, plus a periodic VM matrix on Tier 1. But "passes CI" ≠ "perfect on your specific minimal-image" — file an issue if you find a paper cut.

Built by one person. If you find this useful, a GitHub star is a nice signal. If you find a bug, an issue with the install log is a better one.
